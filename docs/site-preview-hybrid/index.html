Upload Review

Queue an export and let SICFUN review it in the background.

Drop in a PokerStars, Winamax, or GGPoker text export. SICFUN imports the file, identifies the playable hands, accepts the upload quickly, and then returns hand counts, EV-loss review, and opponent reads when the async job completes.

Best Results

The strongest reports come from hands that include the hero hole cards. When the export is missing that information, the review flags the limitation instead of guessing. Large uploads stay on the same HTTP path but run as background jobs so the page stays responsive.

    Scope

    Clear product boundaries keep the review useful.

    Implemented

    Ready for local review sessions

    • Upload supported text hand histories and get a fast async job id back.
    • Review actual versus recommended decisions with EV gaps and hero equity.
    • Return warnings, model source, and opponent reads with the finished review.

    Experimental

    Advanced modeling stays clearly labeled

    • DDRE and related provider paths are still experimental.
    • Multi-player engine progress does not mean SICFUN is a validated 9-max solver.
    • Experimental paths are not presented as proven poker edge.

    Out of Scope

    No live-table play client

    • No real-table integration
    • No action execution against a poker client
    • No promise of one-click real-time play

    Workflow

    How a player or coach uses the current web path.

    Export hands, queue the upload, and start with the biggest gaps when the job completes. The point is simple: spend review time on the decisions that cost the most.

    Queue once

    Supports PokerStars, Winamax, and GGPoker text histories, with an optional hero-name override.

    Wait for completion

    The page polls the local review job until the finished summary, warnings, and decision gaps are ready.

    Keep the trail

    Results stay tied to warnings, model source, and surfaced decision gaps for follow-up work.

    Fit

    Built for disciplined local review. Not for live automation.

    Good fit

    • Solo grinders doing post-session review.
    • Coaches reviewing marked-hand files with students.
    • Teams that want repeatable local evidence, not screenshots alone.

    Not for

    • Real-time table assistance or client control.
    • Files that do not expose the hero hole cards.
    • Anyone expecting the web page to expose every simulator posterior artifact today.
